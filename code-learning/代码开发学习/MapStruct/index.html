MapStruct 转换器使用总结

核心概念

1. 基本用法

2. 使用方式

关键要点

1. 获取实例

  • Mappers.getMapper(XXXXConvert.class) - 获取实现类实例
  • INSTANCE - 静态常量，整个应用只有一个实例

2. 方法签名

  • 只需要写接口方法，不需要写实现
  • 参数自动匹配：相同字段名自动映射
  • 支持多参数：convert(source, param1, param2)

3. 转换方向

  • DO → VO：数据库对象转前端对象
  • VO → DO：前端对象转数据库对象
  • 双向转换：都支持

底层原理

1. 编译时生成

  • MapStruct 在编译时自动生成实现类
  • 不是反射，是静态代码生成
  • 性能等同于手写代码

2. 生成位置

  • 在 target/generated-sources/annotations/ 目录
  • 编译后可以直接查看生成的代码

组织方式

1. 按业务模块划分（推荐）

2. 命名规范

  • 转换器：{业务模块}Convert
  • 方法：convert(), convertList(), convertPage()

使用示例

核心优势

  1. 性能高：编译时生成，运行时直接调用
  2. 类型安全：编译时检查类型匹配
  3. 代码简洁：只需要写方法签名
  4. 自动映射：相同字段名自动匹配
  5. IDE 友好：支持跳转、重构等
