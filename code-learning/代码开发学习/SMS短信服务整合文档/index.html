5. 使用指南

5.1 配置短信渠道

  1. 在管理后台进入”系统管理” -> “短信管理” -> “短信渠道”
  2. 添加新的短信渠道，配置相应的API密钥
  3. 启用渠道状态

5.2 配置短信模板

  1. 进入”短信管理” -> “短信模板”
  2. 创建短信模板，设置模板内容和参数
  3. 关联到对应的短信渠道
  4. 启用模板状态

5.3 发送短信

方式一：直接调用服务

方式二：使用验证码服务

6. 监控和日志

6.1 短信日志查询

  1. 进入”短信管理” -> “短信日志”
  2. 可以查看所有短信发送记录
  3. 支持按手机号、发送状态、时间范围等条件筛选

6.2 日志字段说明

  • 发送状态：0-初始化，1-发送成功，2-发送失败，3-忽略发送
  • 接收状态：0-初始化，1-接收成功，2-接收失败
  • API返回码：短信服务商返回的状态码
  • API返回消息：短信服务商返回的详细信息

8. 扩展开发

8.1 添加新的短信渠道

  1. 在SmsChannelEnum中添加新的渠道类型
  2. 实现对应的SmsClient客户端
  3. 在SmsClientFactoryImpl中注册新的客户端

8.2 自定义短信模板

  1. 在数据库中创建新的模板记录
  2. 设置模板内容和参数
  3. 关联到对应的短信渠道

9. 常见问题

9.1 短信发送失败

  1. 检查短信渠道配置是否正确
  2. 确认短信模板是否已审核通过
  3. 查看短信日志中的错误信息
  4. 检查手机号格式是否正确

9.2 验证码相关问题

  1. 确认验证码未过期（默认5分钟）
  2. 检查验证码是否已被使用
  3. 确认手机号与验证码匹配
  4. 检查发送频率限制

9.3 性能优化

  1. 使用消息队列异步发送短信
  2. 合理设置短信发送频率限制
  3. 定期清理过期的短信日志
  4. 监控短信发送成功率

10. 总结

本项目提供了完整的SMS短信服务解决方案，包括：

  • 多渠道支持：支持阿里云、腾讯云、华为云等主流短信服务商
  • 模板管理：灵活的短信模板配置和管理
  • 验证码功能：完整的验证码发送、校验和使用流程
  • 日志记录：详细的短信发送和接收日志
  • 异步处理：基于消息队列的异步短信发送
  • 防刷机制：验证码发送频率和数量限制
  • 监控管理：完整的短信发送状态监控
