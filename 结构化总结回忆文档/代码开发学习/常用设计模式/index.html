创建型模式

单例模式

用一个静态引用（初始为 null）延迟保存唯一实例；通过两次 null 检查 + 对类对象加锁 + volatile 修饰，实现高效且线程安全的懒加载。

工厂方法模式

定义一个创建对象的接口，让子类决定实例化哪一个类；将对象的创建延迟到子类，解耦产品创建与使用。核心：工厂接口 + 具体工厂实现，客户端通过工厂接口创建产品，无需知道具体产品类。

建造者模式

将复杂对象的构建与表示分离，通过一步步构建最终对象；适用于创建过程复杂、需要灵活组合的场景。核心：抽象建造者定义构建步骤 + 具体建造者实现各步骤 + 分步构建后返回完整产品。

结构型模式

适配器模式

将一个类的接口转换成客户端期望的另一个接口；让原本不兼容的类可以协同工作，通过适配器类进行转换。核心：适配器实现目标接口 + 持有被适配对象 + 在目标方法中调用被适配对象的方法。

装饰器模式

动态地给对象添加额外的职责；通过组合而非继承扩展功能，保持接口一致性，可以灵活地叠加多个装饰器。核心：装饰器实现组件接口 + 持有组件引用 + 先调用组件方法再添加新功能，支持多层嵌套。

代理模式

为其他对象提供一种代理以控制对这个对象的访问；在客户端和目标对象之间增加中间层，实现访问控制、延迟加载等功能。核心：代理实现主题接口 + 持有真实主题引用 + 在请求前后添加控制逻辑，客户端通过代理访问真实对象。

观察者模式

定义对象间一对多的依赖关系，当被观察者状态改变时，所有观察者都会收到通知并自动更新；实现松耦合的事件通知机制。核心：主题维护观察者列表 + 提供注册/移除方法 + 状态改变时遍历调用所有观察者的 update 方法。

状态模式

（待补全）

策略模式

定义一系列算法，把它们封装起来，并使它们可以互相替换；将算法的选择与使用分离，客户端可以动态切换不同的策略。核心：上下文持有策略接口引用 + 提供设置策略方法 + 执行时调用策略接口方法，运行时可切换不同策略实现。

模板方法模式

定义一个操作中算法的骨架，将一些步骤延迟到子类中；子类可以不改变算法结构即可重定义算法的某些特定步骤，实现代码复用。核心：模板方法用 final 定义固定流程 + 抽象方法定义可变步骤 + 子类只需实现抽象方法，算法结构由父类控制。
