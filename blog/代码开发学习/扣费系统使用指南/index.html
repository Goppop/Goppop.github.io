扣费系统使用指南

一、系统概述

扣费系统是算力平台的核心计费模块，支持包月预付费和按小时后付费两种计费方式。系统已完全集成到 pay 模块中。

核心功能

  1. 包月扣费（预付费）：一次性扣除整月费用
  2. 按小时扣费（后付费）：首小时扣费 + 周期续扣
  3. 退款处理：订单取消或失败时的退款
  4. 定时任务：自动执行周期续扣和对账
  5. 同步调用：直接调用扣费服务，实时返回结果

三、API接口说明

1. 创建扣费

接口地址： POST /admin-api/pay/deduction/create

权限： pay:deduction:create

请求示例（包月）：

{
  "orderNo": "ORD20231201001",
  "userId": 1,
  "instanceId": "INS001",
  "deductionType": "MONTHLY_PREPAID",
  "billingType": "MONTHLY",
  "amount": 100.00,
  "unitPrice": 100.00,
  "durationHours": 720,
  "isPeriodic": false
}

请求示例（按小时）：

{
  "orderNo": "ORD20231201002",
  "userId": 1,
  "instanceId": "INS002",
  "deductionType": "HOURLY_POSTPAID",
  "billingType": "HOURLY",
  "amount": 10.00,
  "unitPrice": 10.00,
  "durationHours": 1,
  "isPeriodic": true,
  "periodHours": 1,
  "totalPeriods": -1
}

响应示例：

{
  "code": 0,
  "data": "DED1234567890",
  "msg": "success"
}

2. 退款（支持粗粒度和细粒度）

2.1 按订单号退款（粗粒度）

接口地址： POST /admin-api/pay/deduction/refund-by-order

权限： pay:deduction:refund

适用场景：订单取消、全额退款

请求示例：

{
  "orderNo": "ORD20231201001",
  "reason": "用户申请退款"
}

2.2 按扣费单号退款（细粒度）

接口地址： POST /admin-api/pay/deduction/refund-by-deduction?deductionNo=DED123&reason=原因

权限： pay:deduction:refund

适用场景：升降配、单次扣费退款

请求参数：

  • deductionNo：扣费单号
  • reason：退款原因

2.3 部分退款（细粒度）

接口地址： POST /admin-api/pay/deduction/partial-refund

权限： pay:deduction:refund

适用场景：提前终止、部分退款

请求参数：

  • deductionNo：扣费单号
  • refundAmount：退款金额（必须 > 0 且 <= 原扣费金额）
  • reason：退款原因

请求示例：

POST /admin-api/pay/deduction/partial-refund?deductionNo=DED20250108002&refundAmount=2000.00&reason=提前终止

3. 停止周期扣费（支持粗粒度和细粒度）

3.1 按订单号停止（粗粒度）

接口地址： POST /admin-api/pay/deduction/stop-periodic-by-order?orderNo=ORD20231201002

权限： pay:deduction:stop

适用场景：订单取消、批量停止

3.2 按扣费单号停止（细粒度）

接口地址： POST /admin-api/pay/deduction/stop-periodic-by-deduction?deductionNo=DED123

权限： pay:deduction:stop

4. 查询扣费记录

接口地址： GET /admin-api/pay/deduction/get?id=1

权限： pay:deduction:query

5. 根据订单号查询

接口地址： GET /admin-api/pay/deduction/get-by-order?orderNo=ORD20231201001

权限： pay:deduction:query

6. 分页查询

接口地址： GET /admin-api/pay/deduction/page

权限： pay:deduction:query

请求参数：

  • pageNo: 页码（必填）
  • pageSize: 每页条数（必填）
  • orderNo: 订单号（可选）
  • userId: 用户ID（可选）
  • instanceId: 实例ID（可选）
  • deductionType: 扣费类型（可选）
  • status: 状态（可选）

四、定时任务配置

1. 按小时续扣任务

在 XXL-Job 管理后台添加定时任务：

  • 执行器： pay-executor
  • 任务描述： 按小时续扣任务
  • Cron表达式： 0 * * * * ? （每分钟执行一次）
  • JobHandler： hourlyDeductionJob
  • 路由策略： 第一个
  • 阻塞处理策略： 单机串行

2. 对账任务

  • 执行器： pay-executor
  • 任务描述： 扣费对账任务
  • Cron表达式： 0 0 2 * * ? （每天凌晨2点执行）
  • JobHandler： deductionReconciliationJob
  • 路由策略： 第一个
  • 阻塞处理策略： 单机串行

五、代码集成示例

1. 订单支付成功后扣费

@Service
public class OrderServiceImpl {
    
    @Resource
    private DeductionService deductionService;
    
    @Transactional
    public void handleOrderPaid(OrderDO order) {
        // 1. 构建扣费请求
        DeductionRequest request = new DeductionRequest();
        request.setOrderNo(order.getOrderNo());
        request.setUserId(order.getUserId());
        request.setInstanceId(order.getInstanceId());
        
        if ("MONTHLY".equals(order.getBillingType())) {
            // 包月扣费
            request.setDeductionType("MONTHLY_PREPAID");
            request.setBillingType("MONTHLY");
            request.setAmount(order.getAmount());
            request.setIsPeriodic(false);
        } else {
            // 按小时扣费
            request.setDeductionType("HOURLY_POSTPAID");
            request.setBillingType("HOURLY");
            request.setAmount(order.getUnitPrice()); // 首小时费用
            request.setUnitPrice(order.getUnitPrice());
            request.setIsPeriodic(true);
            request.setPeriodHours(1);
            request.setTotalPeriods(-1); // 无限续扣
        }
        
        // 2. 执行扣费
        DeductionResult result = deductionService.deduct(request);
        
        if (result.getSuccess()) {
            // 扣费成功，更新订单状态
            order.setStatus("PAID");
            order.setDeductionNo(result.getDeductionNo());
            orderMapper.updateById(order);
        } else {
            // 扣费失败，抛出异常回滚订单
            throw new ServiceException("扣费失败：" + result.getFailReason());
        }
    }
}

3. 升配场景（细粒度控制）

@Service
public class InstanceUpgradeService {
    
    @Resource
    private DeductionService deductionService;
    
    @Transactional
    public void upgradeInstance(String orderNo, String instanceId, 
                                 BigDecimal newHourlyPrice) {
        // 1. 查询订单的所有扣费记录
        List<DeductionRecordDO> records = deductionRecordMapper.selectByOrderNo(orderNo);
        
        // 2. 找到旧配置的扣费记录（按创建时间最早）
        DeductionRecordDO oldRecord = records.stream()
            .filter(r -> r.getIsPeriodic() == 1)
            .min(Comparator.comparing(DeductionRecordDO::getCreateTime))
            .orElse(null);
        
        // 3. 停止旧配置的周期扣费（细粒度：按扣费单号）
        if (oldRecord != null) {
            deductionService.stopPeriodicDeductionByDeductionNo(
                oldRecord.getDeductionNo()
            );
        }
        
        // 4. 创建新配置的扣费请求
        DeductionRequest newRequest = DeductionRequest.builder()
            .orderNo(orderNo)
            .userId(userId)
            .instanceId(instanceId)
            .deductionType(DeductionTypeEnum.HOURLY_POSTPAID.getCode())
            .amount(newHourlyPrice)
            .isPeriodic(true)
            .build();
        
        // 5. 执行新配置的首次扣费
        deductionService.deduct(newRequest);
    }
}

4. 提前终止退款（部分退款）

@Service
public class EarlyTerminationService {
    
    @Resource
    private DeductionService deductionService;
    
    @Transactional
    public void terminateEarly(String deductionNo, int usedDays, int totalDays) {
        // 1. 查询扣费记录
        DeductionRecordDO record = deductionRecordMapper.selectByDeductionNo(deductionNo);
        
        // 2. 计算退款金额
        BigDecimal totalAmount = record.getAmount();
        int remainingDays = totalDays - usedDays;
        
        BigDecimal refundAmount = totalAmount
            .multiply(BigDecimal.valueOf(remainingDays))
            .divide(BigDecimal.valueOf(totalDays), 2, RoundingMode.HALF_UP);
        
        // 3. 执行部分退款（细粒度）
        RefundResult result = deductionService.partialRefund(
            deductionNo,
            refundAmount,
            String.format("提前终止，退还剩余%d天费用", remainingDays)
        );
        
        if (result.getSuccess()) {
            log.info("部分退款成功：deductionNo={}, amount={}",
                deductionNo, refundAmount);
        }
    }
}

六、扣费状态流转

PENDING（待扣费）
    ↓
SUCCESS（扣费成功）→ STOPPED（已停止）
    ↓                     ↓
FAILED（扣费失败）     REFUNDED（已退款）

七、幂等性保证

系统通过以下机制保证幂等性：

  1. 幂等键： {orderNo}_{deductionType}_{period}

    • 包月扣费：ORD001_MONTHLY_PREPAID_0
    • 按小时首扣：ORD002_HOURLY_POSTPAID_0
    • 按小时续扣（第2次）：ORD002_HOURLY_POSTPAID_1

  2. 数据库唯一索引： uk_idempotent_key

  3. 重复调用处理：

    • 如果幂等键已存在且状态为SUCCESS，直接返回成功结果
    • 如果幂等键已存在且状态为FAILED，检查重试次数后决定是否重试

八、监控指标

建议监控以下指标：

  1. 扣费成功率： 成功扣费数 / 总扣费数
  2. 余额不足率： 余额不足次数 / 总扣费数
  3. 平均扣费耗时： 扣费接口平均响应时间
  4. 周期扣费延迟： 实际扣费时间 - 计划扣费时间

九、常见问题

1. 余额不足怎么办？

  • 系统会记录失败原因并发送余额不足通知
  • 用户充值后可以手动重试或等待下次续扣
  • 可以配置自动停止实例

2. 扣费失败如何重试？

  • 系统默认最多重试3次
  • 可以通过管理后台手动创建扣费记录重试

3. 如何停止周期扣费？

  • 调用停止周期扣费接口
  • 或者执行退款操作，系统会自动停止周期扣费

4. 对账不平怎么办？

  • 查看对账任务日志，定位具体差异
  • 检查扣费记录表和账户流水表数据
  • 必要时手动调整数据并记录
