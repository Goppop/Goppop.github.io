录价系统设计文档

1. 背景与目标

1.1 背景

当前算力租赁系统中，价格管理存在以下问题：

  • 价格数据分散，缺少统一管理界面
  • 价格模型过于简单，无法支持细粒度定价
  • 缺少价格录入、修改、查询的后台管理功能
  • 无法支持多维度组合计费（CPU规格、GPU规格、地区、附加资源等）

1.2 目标

设计并实现一个完整的录价系统，支持：

  • 细粒度规格定价：CPU规格、GPU规格的精确定价
  • 地区差异定价：不同地区的价格差异
  • 附加资源定价：数据盘、独立IP、收费镜像等
  • 价格管理功能：录入、修改、查询、历史记录
  • 价格计算引擎：自动组合多个维度计算总价

2. 需求分析

2.1 CPU创建计费项

计费项 说明 计费方式
CPU规格（细粒度） 不同CPU核数/型号不同价格 按小时/天/周/月
额外数据盘 按容量（GB）计费 按小时/天/周/月
地区计费 不同地区价格不同 按小时/天/周/月
独立IP 独立IP的额外费用 按小时/天/周/月
收费镜像 某些镜像需要额外收费 按小时/天/周/月

2.2 GPU创建计费项

计费项 说明 计费方式
GPU规格（细粒度） 不同GPU型号/数量不同价格 按小时/天/周/月
额外数据盘 按容量（GB）计费 按小时/天/周/月
地区计费 不同地区价格不同 按小时/天/周/月
独立IP 独立IP的额外费用 按小时/天/周/月
收费镜像 某些镜像需要额外收费 按小时/天/周/月

2.3 功能需求

  1. 价格录入：支持批量录入多种规格的价格
  2. 价格修改：支持价格调整，支持生效时间管理
  3. 价格查询：支持多条件查询、分页查询
  4. 价格历史：记录价格变更历史，支持审计
  5. 价格计算：根据配置自动计算总价

3. 数据模型设计

3.1 核心设计理念

采用组合计费模型，将价格拆分为多个维度：

  • 基础规格价格：CPU/GPU规格的基础价格
  • 地区价格系数：不同地区的价格调整系数
  • 附加资源价格：数据盘、IP、镜像等附加资源价格

3.2 数据表设计

3.2.1 基础规格价格表（tgwk_compute_spec_price）

存储CPU/GPU规格的价格（按地区存储，每个规格+地区+计费方式一条记录）

CREATE TABLE `tgwk_compute_spec_price` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `resource_type` varchar(50) NOT NULL COMMENT '资源类型：CPU/GPU',
  `spec_code` varchar(100) NOT NULL COMMENT '规格编码：如 CPU_4C8G、GPU_RTX4090_1',
  `spec_name` varchar(200) NOT NULL COMMENT '规格名称：如 4核8G、RTX4090×1',
  `cpu_cores` int DEFAULT NULL COMMENT 'CPU核数（CPU资源时使用）',
  `memory_gb` int DEFAULT NULL COMMENT '内存大小GB（CPU资源时使用）',
  `gpu_model` varchar(100) DEFAULT NULL COMMENT 'GPU型号（GPU资源时使用）',
  `gpu_count` int DEFAULT NULL COMMENT 'GPU数量（GPU资源时使用）',
  `region_code` varchar(50) NOT NULL COMMENT '地区编码：如 CN_EAST_1（新增字段）',
  `region_name` varchar(100) DEFAULT NULL COMMENT '地区名称：如 华东1区（新增字段）',
  `billing_method_code` varchar(50) NOT NULL COMMENT '计费方式编码：按量计费/按天/按周/按月',
  `base_price` decimal(10,4) NOT NULL COMMENT '基础价格（元/单位）',
  `currency` varchar(10) NOT NULL DEFAULT 'CNY' COMMENT '货币单位',
  `status` tinyint NOT NULL DEFAULT 1 COMMENT '状态：1=启用，0=禁用',
  `effective_date` datetime DEFAULT NULL COMMENT '生效日期',
  `expire_date` datetime DEFAULT NULL COMMENT '失效日期',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注',
  `creator` varchar(64) DEFAULT '' COMMENT '创建者',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updater` varchar(64) DEFAULT '' COMMENT '更新者',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` bit(1) NOT NULL DEFAULT b'0' COMMENT '是否删除',
  `tenant_id` bigint NOT NULL DEFAULT '0' COMMENT '租户编号',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_spec_region_billing` (`spec_code`, `region_code`, `billing_method_code`, `tenant_id`, `deleted`),
  KEY `idx_resource_type` (`resource_type`),
  KEY `idx_region_code` (`region_code`),
  KEY `idx_billing_method` (`billing_method_code`),
  KEY `idx_status` (`status`),
  KEY `idx_tenant_id` (`tenant_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='算力规格价格表';

设计说明：

  • spec_code：规格唯一编码，如 CPU_4C8G、GPU_RTX4090_1
  • region_code：地区编码，每个规格在不同地区可以有不同价格
  • resource_type：区分CPU和GPU资源
  • billing_method_code：支持多种计费方式
  • base_price：该规格在该地区的直接价格（不再使用系数计算）
  • 唯一索引：spec_code + region_code + billing_method_code，确保同一规格在同一地区同一计费方式只有一条记录

设计变更说明：

  • 去掉地区系数表：因为不同规格在同一地区的倍率可能不同，统一倍率表不适用
  • 在规格价格表中加入地区字段：每个规格+地区+计费方式组合一条记录，价格直接存储
  • 更灵活：支持每个规格在不同地区有完全独立的价格
  • 数据量增加：如果规格多、地区多，数据量会增大，但这是业务需求决定的

  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` bit(1) NOT NULL DEFAULT b'0' COMMENT '是否删除',

设计说明：

    • FIXED：固定价格（如独立IP，每个10元/月）

4. 价格计算引擎设计

4.1 价格计算公式

总价 = 规格地区价格 + 附加资源价格总和

其中：
- 规格地区价格 = 从 tgwk_compute_spec_price 查询（已包含地区信息）
- 附加资源价格 = Σ(附加资源单价 × 数量 × 时长)

设计变更：

  • 不再使用地区系数表：因为不同规格在同一地区的倍率可能不同
  • 直接查询规格+地区的价格：每个规格在不同地区有独立的价格记录

4.2 价格计算流程

1. 查询规格地区价格
   ├─ 根据 resource_type（CPU/GPU）
   ├─ 根据 spec_code（规格编码）
   ├─ 根据 region_code（地区编码）⭐ 新增
   └─ 根据 billing_method_code（计费方式）

2. 获取基础价格
   └─ 直接使用查询到的 base_price（已包含地区差异）

3. 计算附加资源价格
   ├─ 数据盘价格 = 数据盘单价 × 容量(GB) × 时长
   ├─ 独立IP价格 = IP单价 × IP数量 × 时长
   └─ 镜像价格 = 镜像单价 × 时长

4. 计算总价
   └─ 基础价格 + 附加资源价格总和

4.3 价格计算示例

场景：创建GPU实例

  • GPU规格：RTX4090 × 1
  • 地区：华东1区
  • 计费方式：按量计费
  • 时长：10小时
  • 额外数据盘：100GB
  • 独立IP：1个
  • 收费镜像：无

计算过程：

1. 查询规格地区价格
   - spec_code = "GPU_RTX4090_1"
   - region_code = "CN_EAST_1"  ⭐ 直接查询该规格在该地区的价格
   - billing_method_code = "按量计费"
   - base_price = 2.50 元/小时（已包含地区差异）

2. 基础价格计算
   - 10小时总价 = 2.50 × 10 = 25.00 元

3. 附加资源价格计算
   - 数据盘：0.1 元/GB/小时 × 100GB × 10小时 = 100.00 元
   - 独立IP：0.5 元/个/小时 × 1个 × 10小时 = 5.00 元

4. 总价计算
   - 总价 = 25.00 + 100.00 + 5.00 = 130.00 元

说明：

  • 如果 RTX4090 在华南1区（CN_SOUTH_1）的价格是 2.30 元/小时，需要单独配置一条记录
  • 如果 RTX3090 在华东1区的价格是 1.80 元/小时，也需要单独配置
  • 每个规格+地区组合都有独立的价格记录，不再使用统一的地区系数

5. 功能模块设计

5.1 价格管理模块

5.1.1 基础规格价格管理

  • 录入价格：支持批量录入CPU/GPU规格价格
  • 修改价格：支持价格调整，支持生效时间
  • 查询价格：支持按资源类型、规格、计费方式查询
  • 启用/禁用：支持价格启用和禁用

5.1.2 规格地区价格管理

  • 录入价格：为每个规格在不同地区配置价格（规格+地区+计费方式）
  • 批量设置：支持批量设置同一规格在多个地区的价格
  • 查询价格：支持按规格、地区、计费方式查询
  • 价格对比：支持对比同一规格在不同地区的价格差异

5.1.3 附加资源价格管理

  • 数据盘价格：配置数据盘按GB计费的价格
  • 独立IP价格：配置独立IP的价格
  • 镜像价格：配置收费镜像的价格

5.1.4 价格历史查询

  • 历史记录：查看价格变更历史
  • 对比查看：对比价格变更前后的差异
  • 导出记录：支持导出价格历史记录

5.2 价格计算模块

5.2.1 价格预估接口

  • 输入：资源类型、规格、地区、计费方式、时长、附加资源
  • 输出：价格明细、总价

5.2.2 价格计算服务

  • 实时计算：根据配置实时计算价格
  • 缓存优化：缓存常用价格配置，提升性能
