算力平台扣费系统设计方案

一、设计目标

1.1 核心需求

  • 包月（预付费）：一次性扣除整月费用
  • 按小时计费（后付费）：先扣首小时费用，后续每小时自动续扣
  • 资金安全：不重复扣、不多退
  • 操作幂等：同一订单多次调用只生效一次
  • 高可用 & 可对账：支持故障恢复和数据核对
  • 系统解耦：与订单/资源系统解耦，通过消息队列异步通信

1.2 设计原则

  1. 最终一致性：允许短暂的数据不一致，通过补偿机制保证最终一致
  2. 幂等优先：所有扣费操作必须支持幂等
  3. 可追溯性：每笔扣费都有完整的流水记录
  4. 故障隔离：扣费系统故障不影响订单和资源系统

关键设计点：

  1. idempotent_key：保证幂等性，格式：{orderNo}_{deductionType}_{period}，例如：ORD001_MONTHLY_PREPAID_0、ORD002_HOURLY_POSTPAID_1
  2. is_periodic + next_deduction_time：支持周期扣费
  3. status：完整的状态流转，支持失败重试和退款

四、幂等性保证机制

4.1 幂等键设计

格式： {orderNo}_{deductionType}_{period}

示例：

  • 包月扣费：ORD001_MONTHLY_PREPAID_0
  • 按小时首扣：ORD002_HOURLY_POSTPAID_0
  • 按小时第2次：ORD002_HOURLY_POSTPAID_1
  • 按小时第3次：ORD002_HOURLY_POSTPAID_2

4.2 幂等性实现

@Transactional(rollbackFor = Exception.class)
public DeductionResult deduct(DeductionRequest request) {
    // 1. 生成幂等键
    String idempotentKey = generateIdempotentKey(
        request.getOrderNo(), 
        request.getDeductionType(), 
        request.getPeriod()
    );
    
    // 2. 查询是否已存在
    DeductionRecordDO existing = deductionRecordMapper.selectByIdempotentKey(idempotentKey);
    if (existing != null) {
        // 已存在，直接返回结果
        if (DeductionStatusEnum.SUCCESS.equals(existing.getStatus())) {
            return DeductionResult.success(existing);
        }
        // 如果是失败状态，可以重试（在重试次数限制内）
        if (DeductionStatusEnum.FAILED.equals(existing.getStatus()) 
            && existing.getRetryCount() < existing.getMaxRetry()) {
            // 继续执行扣费逻辑
        } else {
            return DeductionResult.fail("扣费已存在且不可重试");
        }
    }
    
    // 3. 创建扣费记录（使用唯一索引保证并发安全）
    try {
        DeductionRecordDO record = createDeductionRecord(request, idempotentKey);
        // 4. 执行扣费
        return executeDeduction(record);
    } catch (DuplicateKeyException e) {
        // 并发情况下，另一个线程已创建，重新查询
        existing = deductionRecordMapper.selectByIdempotentKey(idempotentKey);
        return DeductionResult.success(existing);
    }
}

4.3 数据库唯一索引

UNIQUE KEY `uk_idempotent_key` (`idempotent_key`)

作用：

  • 数据库层面保证幂等键唯一
  • 并发情况下，第二个插入会失败，捕获异常后查询已存在的记录

五、定时任务设计

5.1 按小时续扣任务

任务名称： HourlyDeductionJob

执行频率： 每分钟执行一次

执行逻辑：

@XxlJob("hourlyDeductionJob")
public void hourlyDeduction() {
    // 1. 查询待续扣记录
    // next_deduction_time <= now AND status = SUCCESS AND is_periodic = 1
    List<DeductionRecordDO> records = deductionRecordMapper.selectPendingPeriodicDeductions(
        LocalDateTime.now()
    );
    
    for (DeductionRecordDO record : records) {
        try {
            // 2. 检查实例是否仍在运行（调用资源系统API）
            if (!isInstanceRunning(record.getInstanceId())) {
                // 实例已停止，停止周期扣费
                stopPeriodicDeduction(record);
                continue;
            }
            
            // 3. 计算本次扣费金额（1小时费用）
            BigDecimal amount = calculateHourlyAmount(record);
            
            // 4. 执行扣费
            DeductionRequest request = buildDeductionRequest(record, amount);
            DeductionResult result = deductionService.deduct(request);
            
            if (result.isSuccess()) {
                // 5. 更新下次扣费时间
                updateNextDeductionTime(record, 1); // +1小时
            } else {
                // 扣费失败，记录失败原因
                handleDeductionFailure(record, result);
            }
        } catch (Exception e) {
            log.error("续扣失败，扣费记录ID: {}", record.getId(), e);
            // 记录失败，但不中断其他记录的处理
        }
    }
}

5.2 余额不足处理

private void handleDeductionFailure(DeductionRecordDO record, DeductionResult result) {
    // 1. 更新扣费记录状态
    record.setStatus(DeductionStatusEnum.FAILED);
    record.setFailReason(result.getFailReason());
    deductionRecordMapper.updateById(record);
    
    // 2. 发送余额不足通知（MQ）
    BalanceInsufficientMessage message = new BalanceInsufficientMessage();
    message.setUserId(record.getUserId());
    message.setOrderNo(record.getOrderNo());
    message.setInstanceId(record.getInstanceId());
    message.setRequiredAmount(result.getRequiredAmount());
    messageProducer.sendBalanceInsufficient(message);
    
    // 3. 通知资源系统（可选：自动停止实例）
    if (config.isAutoStopOnInsufficientBalance()) {
        resourceService.stopInstance(record.getInstanceId(), "余额不足");
    }
}

5.3 对账任务

任务名称： DeductionReconciliationJob

执行频率： 每天凌晨2点执行

执行逻辑：

@XxlJob("deductionReconciliationJob")
public void reconciliation() {
    LocalDate yesterday = LocalDate.now().minusDays(1);
    
    // 1. 查询昨天的所有扣费记录
    List<DeductionRecordDO> deductions = deductionRecordMapper.selectByDate(yesterday);
    
    // 2. 查询昨天的所有账户流水
    List<AccountTransactionDO> transactions = accountTransactionService.selectByDate(yesterday);
    
    // 3. 对账：扣费记录金额总和 = 账户流水扣款金额总和
    BigDecimal deductionTotal = deductions.stream()
        .filter(d -> DeductionStatusEnum.SUCCESS.equals(d.getStatus()))
        .map(DeductionRecordDO::getAmount)
        .reduce(BigDecimal.ZERO, BigDecimal::add);
    
    BigDecimal transactionTotal = transactions.stream()
        .filter(t -> TransactionTypeEnum.CONSUME.equals(t.getTransactionType()))
        .map(AccountTransactionDO::getAmount)
        .map(BigDecimal::abs) // 流水表中扣款是负数，取绝对值
        .reduce(BigDecimal.ZERO, BigDecimal::add);
    
    // 4. 比对结果
    if (deductionTotal.compareTo(transactionTotal) != 0) {
        // 对账不平，记录异常
        log.error("对账不平！日期: {}, 扣费总额: {}, 流水总额: {}", 
            yesterday, deductionTotal, transactionTotal);
        // 发送告警通知
        sendReconciliationAlert(yesterday, deductionTotal, transactionTotal);
    }
}

6.2 消息处理

@RabbitListener(queues = "deduction.request.queue")
public void handleDeductionRequest(DeductionRequestMessage message) {
    try {
        // 1. 转换为扣费请求
        DeductionRequest request = convertToRequest(message);
        
        // 2. 执行扣费（幂等）
        DeductionResult result = deductionService.deduct(request);
        
        // 3. 发送扣费结果
        DeductionResultMessage resultMessage = buildResultMessage(result);
        resultProducer.send(resultMessage);
        
    } catch (Exception e) {
        log.error("处理扣费请求失败", e);
        // 发送失败消息或重试
    }
}

7.2 扣费服务实现（核心逻辑）

@Service
@Slf4j
public class DeductionServiceImpl implements DeductionService {
    
    @Resource
    private DeductionRecordMapper deductionRecordMapper;
    
    @Resource
    private UserAccountBalanceService accountBalanceService;
    
    @Resource
    private AccountTransactionService accountTransactionService;
    
    @Override
    @Transactional(rollbackFor = Exception.class)
    public DeductionResult deduct(DeductionRequest request) {
        // 1. 生成幂等键
        String idempotentKey = generateIdempotentKey(request);
        
        // 2. 检查幂等性
        DeductionRecordDO existing = checkIdempotency(idempotentKey);
        if (existing != null && DeductionStatusEnum.SUCCESS.equals(existing.getStatus())) {
            return DeductionResult.success(existing);
        }
        
        // 3. 创建扣费记录
        DeductionRecordDO record = createDeductionRecord(request, idempotentKey);
        
        try {
            // 4. 扣减余额
            BigDecimal balanceBefore = accountBalanceService.getAccount(request.getUserId()).getBalance();
            boolean success = accountBalanceService.reduceBalance(
                request.getUserId(), 
                request.getAmount(), 
                "订单扣费：" + request.getOrderNo()
            );
            
            if (!success) {
                throw new ServiceException("余额不足");
            }
            
            BigDecimal balanceAfter = accountBalanceService.getAccount(request.getUserId()).getBalance();
            
            // 5. 记录账户流水
            String transactionNo = accountTransactionService.recordTransaction(
                AccountTransactionDO.builder()
                    .userId(request.getUserId())
                    .transactionType(TransactionTypeEnum.CONSUME)
                    .amount(request.getAmount().negate()) // 负数表示扣款
                    .balanceBefore(balanceBefore)
                    .balanceAfter(balanceAfter)
                    .relatedOrderNo(request.getOrderNo())
                    .relatedBusinessNo(request.getInstanceId())
                    .businessType(BusinessTypeEnum.INSTANCE_BILLING)
                    .remark("订单扣费：" + request.getOrderNo())
                    .build()
            );
            
            // 6. 更新扣费记录
            record.setStatus(DeductionStatusEnum.SUCCESS);
            record.setDeductionTime(LocalDateTime.now());
            record.setAccountTransactionNo(transactionNo);
            
            // 如果是周期扣费，设置下次扣费时间
            if (request.isPeriodic()) {
                record.setNextDeductionTime(LocalDateTime.now().plusHours(request.getPeriodHours()));
                record.setCurrentPeriod(record.getCurrentPeriod() + 1);
            }
            
            deductionRecordMapper.updateById(record);
            
            return DeductionResult.success(record);
            
        } catch (Exception e) {
            // 扣费失败
            record.setStatus(DeductionStatusEnum.FAILED);
            record.setFailReason(e.getMessage());
            record.setRetryCount(record.getRetryCount() + 1);
            deductionRecordMapper.updateById(record);
            
            return DeductionResult.fail(e.getMessage());
        }
    }
    
    private String generateIdempotentKey(DeductionRequest request) {
        int period = request.isPeriodic() ? request.getCurrentPeriod() : 0;
        return String.format("%s_%s_%d", 
            request.getOrderNo(), 
            request.getDeductionType(), 
            period
        );
    }
    
    private DeductionRecordDO checkIdempotency(String idempotentKey) {
        try {
            return deductionRecordMapper.selectByIdempotentKey(idempotentKey);
        } catch (Exception e) {
            return null;
        }
    }
}

8.1 失败场景处理

失败场景 处理方式 重试策略
余额不足 记录失败原因，发送通知 用户充值后可手动重试

8.2 补偿机制

场景： 扣费成功但后续流程失败（如资源创建失败）

@Transactional(rollbackFor = Exception.class)
public RefundResult refund(String orderNo, String reason) {
    // 1. 查询扣费记录
    List<DeductionRecordDO> records = deductionRecordMapper.selectByOrderNo(orderNo);
    
    BigDecimal totalRefundAmount = BigDecimal.ZERO;
    
    for (DeductionRecordDO record : records) {
        if (!DeductionStatusEnum.SUCCESS.equals(record.getStatus())) {
            continue; // 只退已成功扣费的
        }
        
        // 2. 停止周期扣费（如果是周期扣费）
        if (record.getIsPeriodic() == 1) {
            record.setStatus(DeductionStatusEnum.STOPPED);
            deductionRecordMapper.updateById(record);
        }
        
        // 3. 退款
        BigDecimal refundAmount = record.getAmount();
        accountBalanceService.addBalance(
            record.getUserId(), 
            refundAmount, 
            "订单退款：" + orderNo + "，原因：" + reason
        );
        
        // 4. 记录账户流水
        accountTransactionService.recordTransaction(/* 退款流水 */);
        
        // 5. 更新扣费记录
        record.setStatus(DeductionStatusEnum.REFUNDED);
        deductionRecordMapper.updateById(record);
        
        totalRefundAmount = totalRefundAmount.add(refundAmount);
    }
    
    return RefundResult.success(totalRefundAmount);
}

九、监控和告警

9.1 关键指标

  1. 扣费成功率：成功扣费数 / 总扣费数
  2. 余额不足率：余额不足次数 / 总扣费数
  3. 平均扣费耗时：扣费接口平均响应时间
  4. 周期扣费延迟：实际扣费时间 - 计划扣费时间

9.2 告警规则

  • 扣费成功率 < 95%：发送告警
  • 余额不足率 > 10%：发送告警
  • 对账不平：立即告警
  • 周期扣费延迟 > 5分钟：发送告警

十一、总结

11.1 核心优势

  1. 资金安全：通过乐观锁、幂等键、事务保证
  2. 高可用：异步消息、定时任务、失败重试
  3. 可追溯：完整的扣费记录和账户流水
  4. 可对账：定期对账任务，保证数据一致性
  5. 系统解耦：通过消息队列与订单/资源系统解耦

11.2 注意事项

  1. 幂等键设计：必须包含订单号、扣费类型、周期，保证唯一性
  2. 定时任务频率：建议每分钟执行，避免延迟过大
  3. 余额不足处理：需要与业务方协商，是否自动停止实例
  4. 对账机制：建议每天对账，及时发现数据不一致
