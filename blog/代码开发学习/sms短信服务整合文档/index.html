SMS短信服务

代码开发学习SMS短信服务
PublishedSep 11, 2025
UpdatedNov 21, 2025

SMS短信服务整合文档

概述

本文档详细介绍了项目中SMS短信服务的整合架构、实现方式和使用方法。项目采用了模块化设计，支持多种短信渠道，具备完整的短信发送、模板管理、日志记录和验证码功能。

1. 架构设计

1.1 核心组件

SMS短信服务架构
├── 短信发送服务 (SmsSendService)
├── 短信模板管理 (SmsTemplateService)
├── 短信渠道管理 (SmsChannelService)
├── 短信日志管理 (SmsLogService)
├── 短信验证码服务 (SmsCodeService)
├── 短信客户端工厂 (SmsClientFactory)
└── 消息队列处理 (SmsProducer/SmsConsumer)

2. 核心服务实现

2.1 短信发送服务 (SmsSendService)

主要功能

  • 发送单条短信给管理员用户
  • 发送单条短信给会员用户
  • 异步短信发送处理
  • 短信状态回调处理

核心方法

// 发送短信给管理员
Long sendSingleSmsToAdmin(String mobile, Long userId, 
                         String templateCode, Map<String, Object> templateParams);

// 发送短信给会员
Long sendSingleSmsToMember(String mobile, Long userId, 
                          String templateCode, Map<String, Object> templateParams);

// 异步发送短信
void doSendSms(SmsSendMessage message);

// 处理短信状态回调
void receiveSmsStatus(String channelCode, String text);

发送流程

  1. 验证短信模板和渠道
  2. 校验手机号码
  3. 构建模板参数
  4. 创建发送日志
  5. 发送MQ消息进行异步处理
  6. 更新发送结果

2.2 短信渠道管理 (SmsChannelService)

支持的渠道类型

  • 阿里云短信 (ALIYUN)
  • 腾讯云短信 (TENCENT)
  • 华为云短信 (HUAWEI)
  • 七牛云短信 (QINIU)
  • 钉钉调试 (DEBUG_DING_TALK)

渠道配置

// 渠道配置示例
SmsChannelDO channel = new SmsChannelDO();
channel.setSignature("您的签名");
channel.setCode("ALIYUN");
channel.setApiKey("your_api_key");
channel.setApiSecret("your_api_secret");
channel.setStatus(CommonStatusEnum.ENABLE.getStatus());

2.3 短信模板管理 (SmsTemplateService)

模板类型

  • 验证码短信 (type=1)
  • 通知短信 (type=2)

模板配置示例

// 验证码模板

3. 短信客户端实现

3.1 客户端工厂 (SmsClientFactory)

public interface SmsClient {
    // 发送短信
    SmsSendRespDTO sendSms(Long logId, String mobile, String apiTemplateId,
                          List<KeyValue<String, Object>> templateParams) throws Throwable;
    
    // 解析接收状态
    List<SmsReceiveRespDTO> parseSmsReceiveStatus(String text) throws Throwable;
    
    // 查询短信模板
    SmsTemplateRespDTO getSmsTemplate(String apiTemplateId) throws Throwable;
}

3.2 客户端工厂实现

public class SmsClientFactoryImpl implements SmsClientFactory {
    
    public SmsClient createOrUpdateSmsClient(SmsChannelProperties properties) {
        SmsChannelEnum channelEnum = SmsChannelEnum.getByCode(properties.getCode());
        switch (channelEnum) {
            case ALIYUN: return new AliyunSmsClient(properties);
            case TENCENT: return new TencentSmsClient(properties);
            case HUAWEI: return new HuaweiSmsClient(properties);
            case QINIU: return new QiniuSmsClient(properties);
            case DEBUG_DING_TALK: return new DebugDingTalkSmsClient(properties);
        }
    }
}

4. 消息队列处理

4.1 短信发送生产者 (SmsProducer)

@Component
public class SmsProducer {
    
    public void sendSmsSendMessage(Long logId, String mobile, Long channelId,
                                  String apiTemplateId, List<KeyValue<String, Object>> templateParams) {
        SmsSendMessage message = new SmsSendMessage();
        message.setLogId(logId);
        message.setMobile(mobile);
        message.setChannelId(channelId);
        message.setApiTemplateId(apiTemplateId);
        message.setTemplateParams(templateParams);
        
        // 发送到消息队列
        rocketMQTemplate.asyncSend(SMS_SEND_TOPIC, message);
    }
}

4.2 短信发送消费者 (SmsSendConsumer)

@Component
@RocketMQMessageListener(topic = SMS_SEND_TOPIC, consumerGroup = "sms-send-consumer")
public class SmsSendConsumer implements RocketMQListener<SmsSendMessage> {
    
    @Override
    public void onMessage(SmsSendMessage message) {
        smsSendService.doSendSms(message);
    }
}

5. 使用指南

5.1 配置短信渠道

  1. 在管理后台进入“系统管理” -> “短信管理” -> “短信渠道”
  2. 添加新的短信渠道，配置相应的API密钥
  3. 启用渠道状态

5.2 配置短信模板

  1. 进入“短信管理” -> “短信模板”
  2. 创建短信模板，设置模板内容和参数
  3. 关联到对应的短信渠道
  4. 启用模板状态

5.3 发送短信

方式一：直接调用服务

@Resource
private SmsSendService smsSendService;

// 发送验证码短信
Map<String, Object> params = new HashMap<>();
params.put("code", "123456");
Long logId = smsSendService.sendSingleSmsToMember("11111111111", 1L, 
                                                  "user-sms-login", params);

方式二：使用验证码服务

@Resource
private SmsCodeService smsCodeService;

// 发送验证码
SmsCodeSendReqDTO sendReq = new SmsCodeSendReqDTO();
sendReq.setMobile("11111111111");
sendReq.setScene(SmsSceneEnum.MEMBER_LOGIN.getScene());
sendReq.setCreateIp("127.0.0.1");
smsCodeService.sendSmsCode(sendReq);

5.4 验证码校验

// 校验验证码
SmsCodeValidateReqDTO validateReq = new SmsCodeValidateReqDTO();
validateReq.setMobile("11111111111");
validateReq.setCode("123456");
validateReq.setScene(SmsSceneEnum.MEMBER_LOGIN.getScene());
smsCodeService.validateSmsCode(validateReq);

// 使用验证码（校验通过后标记为已使用）
SmsCodeUseReqDTO useReq = new SmsCodeUseReqDTO();
useReq.setMobile("11111111111");
useReq.setCode("123456");
useReq.setScene(SmsSceneEnum.MEMBER_LOGIN.getScene());
useReq.setUsedIp("127.0.0.1");
smsCodeService.useSmsCode(useReq);

6. 监控和日志

6.1 短信日志查询

  1. 进入“短信管理” -> “短信日志”
  2. 可以查看所有短信发送记录
  3. 支持按手机号、发送状态、时间范围等条件筛选

6.2 日志字段说明

  • 发送状态：0-初始化，1-发送成功，2-发送失败，3-忽略发送
  • 接收状态：0-初始化，1-接收成功，2-接收失败
  • API返回码：短信服务商返回的状态码
  • API返回消息：短信服务商返回的详细信息

8. 扩展开发

8.1 添加新的短信渠道

  1. 在SmsChannelEnum中添加新的渠道类型
  2. 实现对应的SmsClient客户端
  3. 在SmsClientFactoryImpl中注册新的客户端

8.2 自定义短信模板

  1. 在数据库中创建新的模板记录
  2. 设置模板内容和参数
  3. 关联到对应的短信渠道

9. 常见问题

9.1 短信发送失败

  1. 检查短信渠道配置是否正确
  2. 确认短信模板是否已审核通过
  3. 查看短信日志中的错误信息
  4. 检查手机号格式是否正确

9.2 验证码相关问题

  1. 确认验证码未过期（默认5分钟）
  2. 检查验证码是否已被使用
  3. 确认手机号与验证码匹配
  4. 检查发送频率限制

9.3 性能优化

  1. 使用消息队列异步发送短信
  2. 合理设置短信发送频率限制
  3. 定期清理过期的短信日志
  4. 监控短信发送成功率

10. 总结

本项目提供了完整的SMS短信服务解决方案，包括：

  • 多渠道支持：支持阿里云、腾讯云、华为云等主流短信服务商
  • 模板管理：灵活的短信模板配置和管理
  • 验证码功能：完整的验证码发送、校验和使用流程
  • 日志记录：详细的短信发送和接收日志
  • 异步处理：基于消息队列的异步短信发送
  • 防刷机制：验证码发送频率和数量限制
  • 监控管理：完整的短信发送状态监控
