GitHub Pages 部署指南

✅ 已完成的配置

  1. ✅ 已安装 hexo-deployer-git 插件
  2. ✅ 已配置 GitHub 仓库地址：https://github.com/Goppop/Goppop.github.io.git
  3. ✅ 已设置网站 URL：https://goppop.github.io
  4. ✅ 已配置部署分支：main

🚀 部署步骤

第一步：确保 GitHub 仓库已创建

你的仓库地址：https://github.com/Goppop/Goppop.github.io.git

如果仓库是空的，这很正常，Hexo 会自动推送生成的文件。

第二步：配置 Git 认证

方式一：使用 Personal Access Token（推荐）

  1. 生成 Token

    • 访问：https://github.com/settings/tokens
    • 点击 “Generate new token (classic)”
    • 勾选 repo 权限
    • 复制生成的 token

  2. 配置仓库 URL（使用 token）

    在 _config.yml 中，将 deploy 部分改为：

    deploy:
  type: git
  repo: https://你的token@github.com/Goppop/Goppop.github.io.git
  branch: main

    或者使用环境变量（更安全）：

    deploy:
  type: git
  repo: https://${GH_TOKEN}@github.com/Goppop/Goppop.github.io.git
  branch: main

    然后在命令行设置：

    # Windows PowerShell
$env:GH_TOKEN="你的token"

# Windows CMD
set GH_TOKEN=你的token

方式二：使用 SSH（如果已配置 SSH 密钥）

修改 _config.yml：

deploy:
  type: git
  repo: git@github.com:Goppop/Goppop.github.io.git
  branch: main

第三步：生成静态文件

hexo generate
# 或简写
hexo g

第四步：部署到 GitHub

hexo deploy
# 或简写
hexo d

一键部署（生成+部署）

hexo generate --deploy
# 或简写
hexo g -d

📝 完整工作流程

# 1. 清理旧文件（可选）
hexo clean

# 2. 生成静态文件
hexo generate

# 3. 本地预览（可选）
hexo server

# 4. 部署到 GitHub
hexo deploy

⚙️ GitHub Pages 设置

部署完成后，需要在 GitHub 上设置：

  1. 进入仓库设置

    • 访问：https://github.com/Goppop/Goppop.github.io/settings/pages

  2. 配置 Pages

    • Source: 选择 Deploy from a branch
    • Branch: 选择 main，文件夹选择 / (root)
    • 点击 Save

  3. 等待部署

    • 通常几分钟后，你的博客就可以在 https://goppop.github.io 访问了

🔧 常见问题

问题1：部署时提示认证失败

解决方案：

  • 使用 Personal Access Token 替代密码
  • 或者配置 SSH 密钥

问题2：部署后网站显示 404

可能原因：

  • GitHub Pages 设置未配置
  • 分支名称不对（应该是 main 或 master）
  • 等待几分钟让 GitHub 处理

解决方案：

  1. 检查仓库设置中的 Pages 配置
  2. 确认分支名称正确
  3. 等待 5-10 分钟后再访问

问题3：想要自定义域名

  1. 在仓库根目录创建 CNAME 文件（在 source/ 目录下）
  2. 内容为你的域名，例如：blog.yourdomain.com
  3. 在 DNS 中添加 CNAME 记录指向 goppop.github.io

问题4：部署后样式丢失

解决方案：

  • 检查 _config.yml 中的 url 是否正确
  • 运行 hexo clean 后重新生成和部署

📌 重要提示

  1. 不要手动修改 public/ 目录

    • 这个目录是自动生成的，每次 hexo generate 都会重新生成

  2. 部署分支

    • 默认使用 main 分支
    • 如果仓库使用 master 分支，需要修改 _config.yml 中的 branch: master

  3. 源文件管理

    • 建议将整个 Hexo 项目也推送到 GitHub（可以创建另一个仓库，或使用不同的分支）
    • 这样可以在不同电脑上同步你的博客源文件

🎯 推荐工作流程

方案一：单仓库（只部署生成的静态文件）

# 每次更新博客
hexo clean
hexo generate
hexo deploy

方案二：双仓库（推荐）

  1. 仓库1：Goppop.github.io - 存放生成的静态文件（自动部署）
  2. 仓库2：myblog-source - 存放 Hexo 源文件（手动推送）

这样可以：

  • 保留完整的源文件历史
  • 在不同设备上同步
  • 方便版本管理

🔄 更新博客

以后每次更新博客，只需要：

# 1. 添加/修改文章
# （在 source/_posts/ 下添加或修改 .md 文件）

# 2. 生成和部署
hexo clean
hexo generate --deploy
