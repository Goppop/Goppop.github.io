创建型模式

单例模式

private static volatile Singleton instance; // 1. 声明一个静态引用，初始为 null（未初始化）

public static Singleton getInstance() {
    if (instance == null) {                 // 2. 第一次检查：如果没创建，才考虑加锁
        synchronized (Singleton.class) {    // 3. 锁的是 Singleton.class（类对象），不是 instance！
            if (instance == null) {         // 4. 第二次检查：防止多个线程同时通过第2步
                instance = new Singleton(); // 5. 真正创建实例
            }
        }
    }
    return instance;                        // 6. 返回（可能刚创建，也可能早已存在）
}

用一个静态引用（初始为 null）延迟保存唯一实例；通过两次 null 检查 + 对类对象加锁 + volatile 修饰，实现高效且线程安全的懒加载。

工厂方法模式

interface Factory {
    Product createProduct(); // 1. 工厂接口：定义创建产品的方法
}

class ConcreteFactory implements Factory {
    @Override
    public Product createProduct() {
        return new ConcreteProduct(); // 2. 具体工厂：子类决定创建哪个具体产品
    }
}

// 使用：Factory factory = new ConcreteFactory();
//      Product product = factory.createProduct();

定义一个创建对象的接口，让子类决定实例化哪一个类；将对象的创建延迟到子类，解耦产品创建与使用。核心：工厂接口 + 具体工厂实现，客户端通过工厂接口创建产品，无需知道具体产品类。

建造者模式

abstract class Builder {
    protected Product product = new Product(); // 1. 持有产品引用
    public abstract void buildPartA();         // 2. 定义构建各部分的方法
    public abstract void buildPartB();
    public Product getResult() { return product; } // 3. 返回构建完成的产品
}

class ConcreteBuilder extends Builder {
    @Override
    public void buildPartA() { product.setPartA("A"); } // 4. 具体实现各部分构建
    @Override
    public void buildPartB() { product.setPartB("B"); }
}

// 使用：Builder builder = new ConcreteBuilder();
//      builder.buildPartA(); builder.buildPartB();
//      Product product = builder.getResult();

将复杂对象的构建与表示分离，通过一步步构建最终对象；适用于创建过程复杂、需要灵活组合的场景。核心：抽象建造者定义构建步骤 + 具体建造者实现各步骤 + 分步构建后返回完整产品。

适配器模式

class Adapter implements Target {        // 1. 适配器实现目标接口
    private Adaptee adaptee;             // 2. 持有被适配对象的引用
    
    public Adapter(Adaptee adaptee) {
        this.adaptee = adaptee;          // 3. 构造时传入被适配对象
    }
    
    @Override
    public void request() {
        adaptee.specificRequest();       // 4. 将目标接口调用转换为被适配类的方法
    }
}

// 使用：Target target = new Adapter(new Adaptee());
//      target.request(); // 客户端通过目标接口调用，实际执行被适配类的方法

将一个类的接口转换成客户端期望的另一个接口；让原本不兼容的类可以协同工作，通过适配器类进行转换。核心：适配器实现目标接口 + 持有被适配对象 + 在目标方法中调用被适配对象的方法。

装饰器模式

abstract class Decorator implements Component {
    protected Component component;       // 1. 持有被装饰组件的引用
    
    public Decorator(Component component) {
        this.component = component;      // 2. 构造时传入被装饰对象
    }
    
    @Override
    public void operation() {
        component.operation();           // 3. 委托给被装饰组件执行原有功能
    }
}

class ConcreteDecorator extends Decorator {
    @Override
    public void operation() {
        super.operation();               // 4. 先执行被装饰组件的功能
        addedBehavior();                 // 5. 再添加装饰器的新功能
    }
}

// 使用：Component component = new ConcreteDecorator(new ConcreteComponent());
//      component.operation(); // 可以嵌套多个装饰器

动态地给对象添加额外的职责；通过组合而非继承扩展功能，保持接口一致性，可以灵活地叠加多个装饰器。核心：装饰器实现组件接口 + 持有组件引用 + 先调用组件方法再添加新功能，支持多层嵌套。

代理模式

class Proxy implements Subject {
    private RealSubject realSubject;     // 1. 持有真实主题的引用
    
    @Override
    public void request() {
        if (realSubject == null) {
            realSubject = new RealSubject(); // 2. 延迟创建真实对象（可选）
        }
        preRequest();                    // 3. 前置处理（权限检查、日志等）
        realSubject.request();           // 4. 委托给真实主题执行
        postRequest();                   // 5. 后置处理
    }
}

// 使用：Subject subject = new Proxy();
//      subject.request(); // 客户端通过代理访问，代理控制对真实对象的访问

为其他对象提供一种代理以控制对这个对象的访问；在客户端和目标对象之间增加中间层，实现访问控制、延迟加载等功能。核心：代理实现主题接口 + 持有真实主题引用 + 在请求前后添加控制逻辑，客户端通过代理访问真实对象。

观察者模式

class Subject {
    private List<Observer> observers = new ArrayList<>(); // 1. 维护观察者列表
    
    public void attach(Observer observer) {
        observers.add(observer);         // 2. 注册观察者
    }
    
    public void notifyObservers(String message) {
        for (Observer observer : observers) {
            observer.update(message);    // 3. 状态改变时，遍历通知所有观察者
        }
    }
}

// 使用：Subject subject = new Subject();
//      subject.attach(new ConcreteObserver("观察者1"));
//      subject.notifyObservers("状态改变"); // 所有观察者收到通知

定义对象间一对多的依赖关系，当被观察者状态改变时，所有观察者都会收到通知并自动更新；实现松耦合的事件通知机制。核心：主题维护观察者列表 + 提供注册/移除方法 + 状态改变时遍历调用所有观察者的 update 方法。

策略模式

class Context {
    private Strategy strategy;           // 1. 持有策略接口的引用
    
    public void setStrategy(Strategy strategy) {
        this.strategy = strategy;       // 2. 动态设置策略（可运行时切换）
    }
    
    public void executeStrategy() {
        strategy.execute();             // 3. 执行当前策略，无需知道具体实现
    }
}

// 使用：Context context = new Context();
//      context.setStrategy(new ConcreteStrategyA());
//      context.executeStrategy(); // 可以随时切换策略

定义一系列算法，把它们封装起来，并使它们可以互相替换；将算法的选择与使用分离，客户端可以动态切换不同的策略。核心：上下文持有策略接口引用 + 提供设置策略方法 + 执行时调用策略接口方法，运行时可切换不同策略实现。

模板方法模式

abstract class AbstractClass {
    public final void templateMethod() { // 1. 模板方法用 final 固定算法骨架
        step1();                        // 2. 固定步骤：所有子类都执行
        step2();                        // 3. 固定步骤
        step3();                        // 4. 可变步骤：子类必须实现
        step4();                        // 5. 固定步骤
    }
    
    protected abstract void step3();    // 6. 抽象方法：子类实现具体逻辑
}

// 使用：AbstractClass obj = new ConcreteClass();
//      obj.templateMethod(); // 执行固定流程，step3 由子类实现

定义一个操作中算法的骨架，将一些步骤延迟到子类中；子类可以不改变算法结构即可重定义算法的某些特定步骤，实现代码复用。核心：模板方法用 final 定义固定流程 + 抽象方法定义可变步骤 + 子类只需实现抽象方法，算法结构由父类控制。
